# Custom Personas

Add project-specific creative personas here. Each file is a `.md` file with this format:

```markdown
---
id: your_persona_id
name: Your Persona Name
domains: [domain1, domain2]  # or [all]
description: One-line description of the thinking frame
---

## Thinking Frame
What lens does this persona use? What's the core cognitive strategy?

## Approach
1. Step one
2. Step two
3. Step three

## When Most Effective
What kinds of problems does this persona shine on?
```

## Examples of useful custom personas

- **Domain Expert**: A persona that thinks like a senior engineer in your specific domain (e.g., payments, ML infrastructure, mobile)
- **Security Hardener**: Focuses specifically on attack surface and threat modeling
- **Compliance Lens**: Thinks through regulatory and audit requirements
- **Ops/SRE Lens**: Focuses on operability, observability, and incident response
- **Customer Lens**: Specific to your actual customer profile and pain points
